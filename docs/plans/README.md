# Планы фич

Сюда `@planner` сохраняет план каждой фичи как `.md`-файл — чтобы план не терялся, если сессия прервалась, и чтобы видно было, какие фазы уже сделаны.

## Имя файла

```
docs/plans/<YYYY-MM-DD>-<feature-slug>.md
```

Пример: `docs/plans/2026-06-25-stripe-subscriptions.md`

## Статусы фаз

В каждом плане у каждой фазы и у плана целиком есть статус:

| Статус | Значение |
|--------|----------|
| ⬜ Not started | Фаза не начата |
| 🔄 In progress | Сейчас в работе |
| ✅ Done | Фаза закончена и проверена |

## Как это использовать

1. `@planner хочу ...` → planner пишет план и сохраняет в `docs/plans/...`.
2. Реализуете фазу 1 → `@code-reviewer проверь` → попросите: «отметь фазу 1 как done в docs/plans/<файл>».
3. Сессия прервалась? Откройте новый чат и скажите: «продолжи по docs/plans/<файл>» — Claude прочитает файл, увидит какие фазы ✅, и продолжит с первой ⬜/🔄.
4. Все фазы ✅? В конце плана есть секция `## Completion` с готовой git-командой — выполните её (или попросите меня), чтобы зафиксировать готовую работу коммитом.

## Важно

- Planner только **создаёт** файл. Обновлять статус фаз по ходу работы — задача основной модели в чате (у неё есть Edit), не отдельный вызов planner.
- Если план меняется по ходу работы — правьте тот же файл, не создавайте новый на ту же фичу.
- Старые планы можно не удалять — это история того, что и как делалось.
