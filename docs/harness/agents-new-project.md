# Новый проект

Код пишет ИИ. Вы описываете задачу в чате.

## Сначала: скопируйте болванку

Скопируйте всю папку `ai-workspace` → дайте имя проекту → откройте терминал в этой папке и запустите `claude`.

Агенты и код будут жить **в одной корневой папке**.

## Цикл на каждую фичу

| # | Вы | Что происходит |
|---|-----|----------------|
| 1 | `@planner хочу ...` | План в чате |
| 2 | Обсудить план **или** `Реализуй по плану` | Уточняете план **или** ИИ пишет код |
| 3 | `@code-reviewer проверь` | Проверка |
| 4 | `Исправь критичное из ревью` | Правки |

Один чат на всю фичу.

На шаге 2 (реализация) TDD включается автоматически для каждой фичи — это не отдельная просьба, а часть пайплайна `common-development-workflow`.

**Почему иногда AI сразу делает без паузы?**

Planner не умеет создавать файлы — он только составляет план. Но если задача простая (один файл, понятная цель), главная модель видит готовый план и сразу реализует его без остановки. Это нормально и удобно.

| Задача | Что происходит |
|--------|----------------|
| Простая — один файл, ясная цель | Planner планирует → AI сразу делает |
| Сложная — несколько частей, база, API | Planner пишет план → вы читаете → `Реализуй по плану` |

Если хотите увидеть только план и остановиться перед реализацией:
```
@planner хочу ... — только составь план, не реализуй
```

**Planner сохраняет план в файл**, не только в чат: `docs/plans/<дата>-<название>.md`. У каждой фазы там статус ⬜ не начата / 🔄 в работе / ✅ готова — подробности в [docs/plans/README.md](../plans/README.md).

**Сессия прервалась?** Новый чат → `Продолжи по docs/plans/<файл>` — Claude увидит готовые (✅) фазы и продолжит со следующей.

**Крупная задача — попросите отмечать шаги внутри фазы:**

Статус в файле плана — на уровне фаз. Чтобы видеть прогресс по шагам *внутри* текущей фазы, попросите Claude создать список задач:

```
Реализуй фазу 1, создай task
```

Claude создаст список и будет отмечать ✓ каждый шаг. Когда фаза готова целиком — попросите отметить её ✅ в файле плана: `Отметь фазу 1 как done в docs/plans/<файл>`.

## Не знаешь какого агента вызвать?

```
@advisor хочу сделать интернет-магазин с оплатой
```

Advisor скажет кого звать и в каком порядке — коротко и без лишнего.

## Planner или Architect?

| | Planner | Architect |
|--|---------|-----------|
| **Вопрос** | Что делать по шагам? | Как устроить систему целиком? |
| **Когда** | Почти всегда — с него начинайте | Несколько больших частей сразу |
| **Пример** | бот, визитка, форма | магазин + оплата + база + email |

**Planner** сам пишет в плане: какие папки, файлы, шаги. Для простых задач **architect не нужен**.

**Architect** — **до** planner, если проект сложный:

```
1. @architect как устроить: интернет-магазин — каталог, корзина, оплата, письма на email
2. @planner план по архитектуре выше
3. Обсудите план — или: «Реализуй по плану»
4. @code-reviewer проверь
```

**Не нужен architect**, если одна простая вещь:
- сайт-визитка
- телеграм-бот
- одна форма, один скрипт

## Пример: сайт-визитка

```
1. @planner хочу сайт-визитку: имя, описание, контакты, кнопка в Telegram
2. Обсудите план — или сразу:
   «Добавь в план тёмную тему»
   «Реализуй по составленному плану»
3. @code-reviewer проверь
4. Исправь критичное
```

## Пример: телеграм-бот

```
1. @planner хочу телеграм-бота: каждый день в 9:00 присылать погоду
2. Обсудите план — или сразу:
   «Сделай город настраиваемым в плане»
   «Реализуй по плану»
3. @code-reviewer проверь
4. @security-reviewer проверь   ← бот с токеном
5. Исправь критичное
```

## Пример: интернет-магазин (с architect)

```
1. @architect как устроить: каталог, корзина, оплата картой, письмо после заказа
2. @planner план по этой архитектуре
3. Обсудите план — или: «Реализуй по плану»
4. @code-reviewer проверь
5. @security-reviewer проверь   ← оплата и личные данные
6. Исправь критичное
```

## Другие агенты (по ситуации)

| Ситуация | Агент | Пример |
|----------|-------|--------|
| TypeScript / JavaScript код | typescript-reviewer | `@typescript-reviewer проверь` |
| Python код | python-reviewer | `@python-reviewer проверь .py файлы` |
| Формы, пароли, файлы | security-reviewer | `@security-reviewer проверь регистрацию` |
| Хотите вызвать TDD вручную, отдельно от обычного цикла (например, баг без полного плана) | tdd-guide | `@tdd-guide баг: скидка считается неверно` |
| Код раздулся, нечитаемый | code-simplifier | `@code-simplifier упрости, ничего не ломая` |
| Что-то не работает без ошибок | silent-failure-hunter | `@silent-failure-hunter молча не сохраняет` |
| Мусор, дублирование в коде | refactor-cleaner | `@refactor-cleaner убери лишнее` |
| Нужен desktop GUI на PySide6 | qt-ui-pipeline-bootstrapper | `@qt-ui-pipeline-bootstrapper настрой пайплайн .ui` |
| Есть Qt/Tkinter GUI, нужна веб-версия | qt-html-mirror-builder | `@qt-html-mirror-builder сделай HTML версию GUI` |
| Проект работает, нужен README | docs-writer | `@docs-writer README: как запустить` |
| Документация устарела после правок | doc-updater | `@doc-updater обнови карту проекта` |

**docs-writer vs doc-updater:** docs-writer пишет README/гайды с нуля для людей. doc-updater обновляет уже существующую документацию и карту файлов проекта после правок в коде.

Остальные агенты — в [шпаргалке](./agents-cheatsheet.md).
