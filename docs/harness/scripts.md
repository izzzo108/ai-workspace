# scripts/ — Claude + Codex + GLM

Связки скриптов, где несколько LLM работают вместе над одной задачей, без ручного
копирования текста между чатами. Движки на выбор: **Claude**, **Codex** и **GLM**
(последний — через `opencode`; показывается в выборе, только если `opencode` установлен).

- **План-ревью** (`plan-review.sh`, `plan-review-live.py`) — начинающий составляет план,
  отвечающий критикует, начинающий дорабатывает. Описан ниже.
- **Мозговой штурм** (`brainstorm-live.py`) — многораундовый дебат с подведением итогов
  в нужном вам виде. Описан в конце файла.

Общая логика (движки, выбор моделей, живой веб-UI) вынесена в `scripts/agents_common.py`,
который импортируют оба `*-live.py`.

> **Все скрипты запускаются через Git Bash** — именно там в PATH лежат CLI
> `claude` / `codex` / `opencode`, которыми пользуются скрипты.

---

# План-ревью

Получить план от одной LLM, отдать его на критику другой и получить доработанную
финальную версию — как code review, но для плана.

## Что делают скрипты

1. **Начинающий составляет план** по вашей задаче → `docs/Plans/<дата>_<время>_plan.md`.
2. **Отвечающий критикует план** — слабые места, риски, недостающие шаги →
   `docs/Plans/<дата>_<время>_critique.md`.
3. **Начинающий дорабатывает план** с учётом критики (принимает/отклоняет замечания,
   объясняя почему) → `docs/Plans/<дата>_<время>_plan-final.md`.

Вы выбираете **движок и модель для каждой роли** (начинающий / отвечающий). Комбинации
любые: Claude+Codex, Codex+GLM, два Claude и т.п. Контекст между шагами передаётся моделям
прямо в промпте, а все файлы пишет сам скрипт — поэтому в любой роли одинаково работают
все три движка. После каждого шага скрипт **проверяет на диске**, что файл появился.

## Два варианта

| Файл | Когда использовать |
|------|---------------------|
| `scripts/plan-review.sh` | Вывод прямо в терминал Git Bash |
| `scripts/plan-review-live.py` | Живая веб-страница `http://127.0.0.1:8765` со сворачиваемыми блоками по шагам; дополнительно сохраняет `docs/Plans/<дата>_<время>_conversation.html` (статичную копию хода) |

Для каждой роли выбираются **движок → модель → глубина размышления** (в том виде, как её
даёт движок).

Модели:
- **Claude** — `sonnet` / `opus` / `haiku` / `fable`;
- **Codex** — семейство `gpt-5.6` (`sol` — флагман, `terra` — баланс, `luna` — быстрая/дешёвая),
  а также `gpt-5.5` / `gpt-5.4` / `gpt-5.4-mini`. Live-версия подтягивает актуальный список
  из кэша твоего codex CLI, поэтому новые модели появляются в выборе автоматически;
- **GLM** — `glm-5.2` / `glm-5.1` / `glm-4.7` / `glm-5-turbo` / `glm-4.5-air`.

Глубина размышления задаётся так, как её предоставляет каждый движок:
- **Claude** — `--effort`: `low` / `medium` / `high` / `xhigh` / `max`;
- **Codex** — уровень зависит от модели: у `gpt-5.6 Sol/Terra` до `max`/`ultra`, у `Luna` до
  `max`, у `5.5`/`5.4` до `xhigh`;
- **GLM** — provider-specific `--variant`: у `glm-5.2` — `high` / `max`, у остальных
  размышление включено по умолчанию без градаций.

## Запуск (через Git Bash)

```bash
# Терминальная версия
./scripts/plan-review.sh "хочу телеграм-бота с уведомлениями раз в день"

# Live-версия (живой просмотр в браузере)
python scripts/plan-review-live.py "хочу телеграм-бота с уведомлениями раз в день"
```

# Мозговой штурм (`brainstorm-live.py`)

Многораундовый дебат **трёх LLM на выбор** (Claude / Codex / GLM) по вашей теме с
подведением итогов в нужном вам виде — чтобы «продумать» задачу несколькими независимыми
моделями и получить готовый результат. Живой просмотр переписки в браузере
(`http://127.0.0.1:8765`).

> Простое пошаговое объяснение для не-программиста — в [brainstorm-guide.md](./brainstorm-guide.md).

## Как это работает

1. **Бриф.** Скрипт спрашивает:
   - **Что обдумать** — задача (обязательно; можно передать аргументом запуска);
   - несколько **необязательных уточнений** (контекст, что уже известно, критерии успеха,
     акценты, что вне рамок) — дают моделям больше контекста; любой пункт можно пропустить
     (Enter);
   - **Как назвать папку** — короткое имя. Результаты лягут в
     `docs/brainstorm/<ваше-имя>_ДД_ММ_ГГГГ/` (дата добавляется автоматически).
2. **Три слота — выбор движка, модели и глубины размышления для каждого:**
   - **Начинающий** — предлагает идеи;
   - **Отвечающий** — критикует / оппонирует;
   - **Подводящий итоги** — собирает финальный результат в запрошенном виде.

   Любой слот — любой из движков (Claude / Codex / GLM). Комбинации любые: «Codex начинает /
   Claude отвечает», два Claude, GLM+Codex, кто угодно подводит итоги. Список моделей и
   уровни размышления — см. раздел «План-ревью» выше.
3. **Число раундов** — вводите вручную (1 раунд = «предложение + критика»).
4. **Диалог** идёт `начинающий → отвечающий` заданное число раундов; каждый ход видит всю
   накопленную переписку. В браузере — только содержательные блоки, без сырого JSON и
   служебного шума (Claude — оранжевый, Codex — синий, GLM — зелёный, итог — золотой).
5. **Подводящий итоги** читает всю переписку и оформляет итог как **красивую
   самодостаточную HTML-страницу** для удобного чтения — с простыми визуализациями
   (встроенный SVG: столбики, шкалы, таймлайн), когда это уместно для содержания.

Всё сохраняется на диск самим скриптом (модели ничего не пишут, контекст передаётся им
прямо в промпте) — поэтому в любом слоте одинаково работают все три движка.

## Что сохраняется

Каждый запуск создаёт папку `docs/brainstorm/<ваше-имя>_ДД_ММ_ГГГГ/` (имя задаёте вы,
дата добавляется сама; повторный запуск с тем же именем не перезаписывает прошлый):

- `result.html` — итог **всегда** как красивая, готовая к открытию страница (с графиками
  на встроенном SVG, где уместно). Скрипт извлекает чистый HTML из ответа, а если модель
  вернула markdown — сам собирает из него аккуратную страницу;
- `conversation.html` — статичная копия всего диалога, открывается в браузере без
  запущенного сервера;
- `transcript.md` — задача + все раунды + финал (markdown).

## Запуск (через Git Bash)

```bash
python scripts/brainstorm-live.py "приложение для скачивания картинок с сайта по категориям"
# или без аргумента — задачу спросят в брифе:
python scripts/brainstorm-live.py
```

## Требования (для всех скриптов)

- Запуск **через Git Bash** (там доступны нужные CLI).
- Залогиненные CLI `claude` и `codex` в PATH.
- Для GLM — установленный `opencode` с провайдером `zai-coding-plan` (модели `glm-*`).
  Если `opencode` не установлен, GLM просто не появится в выборе — остальное работает.
  На Windows npm ставит `opencode` как `.cmd`-обёртку — скрипты сами находят настоящий
  `.exe`, ничего настраивать не нужно.
- Python 3 (только стандартная библиотека, без внешних зависимостей).

## Зачем это нужно

Второе (и третье) независимое мнение о плане или идее ловит слепые пятна одной модели.
Полезно перед стартом крупной фичи, где цена ошибки выше, чем время на ещё один прогон.
