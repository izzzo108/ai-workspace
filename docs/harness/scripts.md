# scripts/ — Claude + Codex план-ревью

Два способа получить план от Claude, отдать его на критику Codex и получить
доработанную финальную версию — без ручного копирования текста между чатами.

## Что делают скрипты

Оба скрипта реализуют один и тот же 3-шаговый пайплайн:

1. **Claude составляет план** по вашей задаче (используя `planner.md` как инструкцию)
   и сохраняет его в `docs/Plans/<дата>_<время>_plan.md`.
2. **Codex критикует план** — ищет слабые места, риски, недостающие шаги — критика
   сохраняется в `docs/Plans/<дата>_<время>_critique.md`.
3. **Claude дорабатывает план** с учётом критики (принимает или отклоняет замечания,
   объясняя почему) и сохраняет финал в `docs/Plans/<дата>_<время>_plan-final.md`.

После каждого шага скрипт **проверяет на диске**, что файл реально появился — не
верит на слово ответу модели.

## Два варианта

| Файл | Когда использовать |
|------|---------------------|
| `scripts/plan-review.sh` | Bash / Git Bash / Linux, вывод прямо в терминал |
| `scripts/plan-review-live.py` | Кроссплатформенно (нужен только Python), поднимает локальную веб-страницу `http://127.0.0.1:8765` с живым потоком событий (Claude — оранжевым, Codex — синим), сворачиваемые блоки по шагам |

Оба спрашивают интерактивно, какую модель Claude (`sonnet` / `opus` / `haiku` / `fable`)
и какую модель Codex (`gpt-5.5` / `gpt-5.4-mini` / `gpt-5.3-codex`) использовать.

## Требования

- Установленные и залогиненные CLI `claude` и `codex` в PATH.
- Для `.py`-варианта — Python 3 (стандартная библиотека, без внешних зависимостей).
- Для `.sh`-варианта — один раз выполнить `chmod +x scripts/plan-review.sh`.

## Запуск

```bash
# Bash-версия
./scripts/plan-review.sh "хочу телеграм-бота с уведомлениями раз в день"

# Python-версия (живой просмотр в браузере)
python scripts/plan-review-live.py "хочу телеграм-бота с уведомлениями раз в день"
```

## Зачем это нужно

Второе, независимое мнение (Codex) о плане Claude ловит слепые пятна одной модели —
похоже на code review, но для плана, а не для кода. Скрипт полезен перед стартом
крупной фичи, где цена ошибки в плане выше, чем время на второй прогон.
