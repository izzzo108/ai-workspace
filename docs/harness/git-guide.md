# Git — просто и по шагам

`.git` — папка, где хранится история изменений проекта. Создаётся один раз, дальше пополняется только когда вы сами это просите.

## Откуда берётся `.git`

`setup.bat` сам создаёт `.git`, если его ещё нет — ничего делать не нужно.

Если папки `.git` всё же нет (например, `setup.bat` не запускали), создать вручную:
```
git init
```

## Главное правило

Git **не** записывает изменения сам и не тратит ничего "в фоне". Запись происходит только после трёх команд, которые вы выполняете руками (или просите меня выполнить).

## 3 команды, которые нужны почти всегда

| # | Команда | Что делает |
|---|---------|------------|
| 1 | `git status` | Показывает, что изменилось с прошлого раза |
| 2 | `git add .` | Помечает все изменения как "готовы к сохранению" |
| 3 | `git commit -m "что сделал"` | Сохраняет снимок проекта с описанием |

Типичный цикл после куска работы:
```
git status
git add .
git commit -m "добавил экран настроек"
```

## Когда коммитить

Коммитьте после каждого осмысленного шага: фича готова, баг исправлен, рефактор закончен. Не нужно коммитить после каждой строчки — но и не нужно копить недели изменений в один коммит.

## Посмотреть историю

```
git log --oneline
```
Покажет список всех сохранённых снимков (коммитов), от новых к старым.

## Откатиться к прошлой версии

Если что-то поломалось и нужно вернуть файл к последнему сохранённому состоянию:
```
git checkout -- имя_файла
```

Если нужно вернуть **весь проект** к последнему коммиту (все несохранённые изменения потеряются — будьте осторожны):
```
git reset --hard
```

## Это всё локально

Пока вы не подключили удалённый репозиторий (например, GitHub) и не сделали `git push` — все коммиты живут только на вашем компьютере. Никуда не отправляются и никем не видны.
